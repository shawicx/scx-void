# 🤖 **AI Code Agent - Basic Project Rules**

以下是适用于此项目的基本规则和指导原则。

---

## **核心原则**

1. **保持现有功能完整性**
   - 除非明确要求，否则不要修改现有功能
   - 保持项目原有的构建流程和运行方式

2. **代码风格统一**
   - 遵循项目现有的代码风格和命名规范
   - 保持文件和目录结构的一致性

3. **依赖管理**
   - 使用项目指定的包管理器
   - 不要随意降级依赖版本
   - 新增依赖需要考虑兼容性

4. **最小化修改**
   - 只做必要的修改
   - 避免影响不相关的功能模块

---

## **项目特定规则**

- 遵循项目的技术栈约定
- 保持现有的配置文件格式
- 不要引入不兼容的 API 或功能
- 测试新添加的功能

---

## **输出要求**

- 提供清晰的代码实现
- 必要时说明修改原因
- 避免冗余的解释性文本
