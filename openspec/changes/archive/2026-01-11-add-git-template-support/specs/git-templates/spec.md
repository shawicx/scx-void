## 新增需求

### 需求：Git 模板选择
系统应提供用户从预定义注册表选择 Git 模板或指定自定义 Git 仓库 URL 进行项目初始化的能力。

#### 场景：交互式 Git 模板选择
- **给定**：用户运行 `scx-void project init` 且未指定源
- **当**：用户从源选择菜单中选择"Git 仓库模板"
- **那么**：系统应显示预定义 Git 模板列表，包含名称和描述
- **并且**：系统应包含指定自定义 Git 仓库 URL 的选项

#### 场景：使用预定义模板的命令行 Git 模板选择
- **给定**：用户运行 `scx-void project init --source=git --template=node-ts-cli --name=my-project`
- **当**：命令执行
- **那么**：系统应使用由"node-ts-cli"标识的预定义 Git 模板
- **并且**：系统应从该模板创建名为"my-project"的项目

#### 场景：使用自定义 URL 的命令行 Git 模板选择
- **给定**：用户运行 `scx-void project init --source=git --url=https://github.com/user/repo.git --name=my-project`
- **当**：命令执行
- **那么**：系统应从指定 URL 克隆仓库
- **并且**：系统应从克隆的仓库创建名为"my-project"的项目

### 需求：Git 仓库克隆
系统应将 Git 仓库克隆到临时目录并将模板文件提取到项目目录。

#### 场景：成功的 Git 仓库克隆
- **给定**：有效的 Git 仓库 URL
- **当**：系统启动克隆操作
- **那么**：系统应将仓库克隆到临时目录
- **并且**：系统应默认使用浅克隆（`--depth 1`）以提高速度
- **并且**：系统应验证克隆成功

#### 场景：克隆特定分支的 Git 仓库
- **给定**：Git 仓库 URL 和通过 `--branch=develop` 指定的分支名称
- **当**：系统启动克隆操作
- **那么**：系统应克隆指定的分支而不是默认分支
- **并且**：系统应验证分支存在

#### 场景：使用模板路径的 Git 仓库克隆
- **给定**：Git 仓库 URL 和通过 `--template-path=/templates/base` 指定的模板路径
- **当**：系统启动克隆操作
- **那么**：系统应克隆仓库
- **并且**：系统应仅从指定的子目录路径提取文件

#### 场景：Git 仓库克隆失败
- **给定**：无效或无法访问的 Git 仓库 URL
- **当**：系统尝试克隆仓库
- **那么**：系统应返回 `GitCloneError` 并附带描述性消息
- **并且**：系统不应创建项目目录

### 需求：Git 模板验证
系统应在尝试克隆之前验证 Git 仓库 URL、分支名称和模板路径。

#### 场景：有效的 Git URL 验证
- **给定**：Git URL，如 `https://github.com/user/repo.git` 或 `git@github.com:user/repo.git`
- **当**：系统验证 URL
- **那么**：系统应接受 URL 为有效

#### 场景：无效的 Git URL 验证
- **给定**：无效的 URL，如 `not-a-url` 或 `http://example.com`
- **当**：系统验证 URL
- **那么**：系统应拒绝 URL
- **并且**：系统应返回 `InvalidGitUrl` 错误

#### 场景：Git 分支存在性检查
- **给定**：Git 仓库 URL 和分支名称
- **当**：系统检查分支是否存在
- **那么**：系统应验证分支在仓库中存在
- **并且**：如果分支不存在，系统应返回 `GitBranchNotFound` 错误

### 需求：临时文件管理
系统应使用临时目录来存放克隆的仓库，并在项目创建后自动清理。

#### 场景：自动清理临时目录
- **给定**：Git 模板克隆到临时目录
- **当**：模板文件提取到项目目录
- **那么**：系统应自动删除临时目录
- **并且**：系统在项目创建后不应留下任何临时文件

#### 场景：错误时的临时目录清理
- **给定**：Git 模板克隆到临时目录
- **当**：项目创建期间发生错误
- **那么**：系统应自动删除临时目录
- **并且**：系统不应留下任何临时文件

### 需求：模板注册表
系统应维护预定义 Git 模板的注册表，包含标识符、名称、描述和仓库 URL。

#### 场景：检索所有预定义模板
- **给定**：系统有预定义模板的注册表
- **当**：系统检索所有模板
- **那么**：系统应返回所有可用模板的列表
- **并且**：每个模板应包含 id、display_name、description、repository_url、default_branch 和 project_type

#### 场景：通过 ID 检索特定模板
- **给定**：模板 ID "node-ts-cli" 存在于注册表中
- **当**：系统通过 ID 检索模板
- **那么**：系统应返回具有该 ID 的模板
- **并且**：模板应包含所有必需信息

#### 场景：不存在的模板 ID
- **给定**：模板 ID "non-existent" 不存在于注册表中
- **当**：系统尝试通过 ID 检索模板
- **那么**：系统应返回 `None` 或指示模板未找到的错误

### 需求：向后兼容性
系统应与现有本地模板功能保持向后兼容性。

#### 场景：本地模板创建不变
- **给定**：用户运行 `scx-void project init` 且没有任何 Git 相关参数
- **当**：用户从源选择菜单中选择"本地模板"
- **那么**：系统应使用现有的本地模板功能
- **并且**：行为应与以前的版本相同

#### 场景：使用现有命令的本地模板
- **给定**：用户运行 `scx-void project init` 并且仅交互式提供项目名称
- **当**：命令执行
- **那么**：系统应默认为本地模板工作流
- **并且**：不应执行任何 Git 相关操作

### 需求：Git 安装检查
系统应在尝试 Git 操作之前验证系统上是否安装了 Git。

#### 场景：Git 已安装
- **给定**：系统上安装了 Git
- **当**：系统检查 Git 安装
- **那么**：系统应检测到 Git 可用
- **并且**：系统应继续 Git 操作

#### 场景：Git 未安装
- **给定**：系统上未安装 Git
- **当**：系统检查 Git 安装
- **那么**：系统应返回 `GitNotInstalled` 错误
- **并且**：错误消息应包含如何安装 Git 的说明

### 需求：混合模式交互
系统应支持混合模式，其中一些参数通过命令行参数提供，其他参数通过交互方式收集。

#### 场景：指定 Git 源，交互式选择模板
- **给定**：用户运行 `scx-void project init --source=git`
- **当**：命令执行
- **那么**：系统应跳过源选择菜单
- **并且**：系统应交互式显示 Git 模板选择菜单

#### 场景：指定模板，交互式输入项目名称
- **给定**：用户运行 `scx-void project init --template=node-ts-cli`
- **当**：命令执行
- **那么**：系统应跳过模板选择
- **并且**：系统应交互式提示项目名称

## 修改需求

### 需求：项目初始化
系统应支持从本地模板和 Git 仓库初始化项目，并允许用户选择模板源。

#### 场景：从 Git 模板交互式初始化项目
- **给定**：用户运行 `scx-void project init`
- **当**：用户从源菜单选择"Git 仓库模板"
- **并且**：用户选择预定义 Git 模板
- **并且**：用户输入项目名称
- **那么**：系统应克隆 Git 仓库
- **并且**：系统应将模板文件提取到项目目录
- **并且**：系统应成功创建项目

#### 场景：从本地模板初始化项目（现有行为）
- **给定**：用户运行 `scx-void project init`
- **当**：用户从源菜单选择"本地模板"
- **并且**：用户选择项目类型
- **并且**：用户输入项目名称
- **那么**：系统应使用本地模板创建项目
- **并且**：系统不应执行任何 Git 操作

#### 场景：使用完整命令行参数初始化项目
- **给定**：用户运行 `scx-void project init --source=git --template=node-ts-cli --branch=develop --template-path=/base --name=my-project`
- **当**：命令执行
- **那么**：系统应在没有任何交互提示的情况下创建项目
- **并且**：所有指定的参数应用于项目创建

### 需求：CLI 参数
CLI 应支持用于指定 Git 模板源、模板标识符、仓库 URL、分支和模板路径的可选参数。

#### 场景：Git 源参数
- **给定**：用户提供 `--source=git`
- **当**：CLI 解析
- **那么**：系统应将模板源设置为 Git

#### 场景：模板 ID 参数
- **给定**：用户提供 `--template=node-ts-cli`
- **当**：CLI 解析
- **那么**：系统应使用 ID 为"node-ts-cli"的预定义模板

#### 场景：自定义 URL 参数
- **给定**：用户提供 `--url=https://github.com/user/repo.git`
- **当**：CLI 解析
- **那么**：系统应使用指定的自定义 Git 仓库 URL

#### 场景：分支参数
- **给定**：用户提供 `--branch=develop`
- **当**：CLI 解析
- **那么**：系统应克隆指定的分支而不是默认分支

#### 场景：模板路径参数
- **给定**：用户提供 `--template-path=/templates/base`
- **当**：CLI 解析
- **那么**：系统应仅从指定的子目录路径提取文件
